authorize:revokeAccess:forceOAuth:inApp:

Starts authorization process. Otherwise Mobile Safari will be opened for access request.

+ (void)authorize:(NSArray *)permissions revokeAccess:(BOOL)revokeAccess forceOAuth:(BOOL)forceOAuth inApp:(BOOL)inApp

Parameters

permissions

Array of permissions for your applications. All permissions you can

revokeAccess

If YES, user will allow logout (to change user)

forceOAuth

If YES, SDK will use only oauth authorization through mobile safari. Otherwise, it will try to authorize through VK application

inApp

If YES, SDK will try to open modal window with webview to authorize. This method strongly not recommended as user should enter his account data in your application. For use modal view add VKSdkResources.bundle to your project.
